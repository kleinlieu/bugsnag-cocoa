bsg_kscrash_install

BSG_KSCrashType bsg_kscrash_install(
    const char *const crashReportFilePath, 
    const char *const recrashReportFilePath, 
    const char *stateFilePath, 
    const char *crashID);
Parameters
crashReportFilePath

The file to store the next crash report to.

recrashReportFilePath

If the system crashes during crash handling, store a second, minimal report here.

stateFilePath

File to store persistent state in.

crashID

The unique identifier to assign to the next crash report.

Return Value

The crash types that are being handled.

Discussion

Install the crash reporter. The reporter will record the next crash and then terminate the program.

bsg_kscrash_reportUserException

void bsg_kscrash_reportUserException(
    const char *name,
    const char *reason, 
    const char *language, 
    const char *lineOfCode, 
    const char *stackTrace, 
    bool terminateProgram);
Parameters
name

The exception name (for namespacing exception types).

reason

A description of why the exception occurred.

language

A unique language identifier.

lineOfCode

A copy of the offending line of code (NULL = ignore).

stackTrace

JSON encoded array containing stack trace information (one frame per array entry). The frame structure can be anything you want, including bare strings.

terminateProgram

If true, do not return from this function call. Terminate the program instead.

Discussion

Report a custom, user defined exception. This can be useful when dealing with scripting languages.

If terminateProgram is true, all sentries will be uninstalled and the application will terminate with an abort().

bsg_kscrash_setDeadlockWatchdogInterval

void bsg_kscrash_setDeadlockWatchdogInterval(
    double deadlockWatchdogInterval);
Discussion

Set the maximum time to allow the main thread to run without returning. If a task occupies the main thread for longer than this interval, the watchdog will consider the queue deadlocked and shut down the app and write a crash report.

Warning: Make SURE that nothing in your app that runs on the main thread takes longer to complete than this value or it WILL get shut down! This includes your app startup process, so you may need to push app initialization to another thread, or perhaps set this to a higher value until your application has been fully initialized.

0 = Disabled. Default: 0

bsg_kscrash_setHandlingCrashTypes

BSG_KSCrashType bsg_kscrash_setHandlingCrashTypes(
    BSG_KSCrashType crashTypes);
Parameters
crashTypes

The crash types to handle.

Return Value

The crash types that are now behing handled. If BSG_KSCrash has been installed, the return value represents the crash sentries that were successfully installed. Otherwise it represents which sentries it will attempt to activate when BSG_KSCrash installs.

Discussion

Set the crash types that will be handled. Some crash types may not be enabled depending on circumstances (e.g. running in a debugger).

bsg_kscrash_setIntrospectMemory

void bsg_kscrash_setIntrospectMemory(
    bool introspectMemory);
Discussion

If true, introspect memory contents during a crash. Any Objective-C objects or C strings near the stack pointer or referenced by cpu registers or exceptions will be recorded in the crash report, along with their contents.

Default: false

bsg_kscrash_setSearchQueueNames

void bsg_kscrash_setSearchQueueNames(
    bool shouldSearchQueueNames);
Discussion

If true, search for dispatch queue names where appropriate. Queue name searching is not async-safe, and so comes with the risk of timing out and panicking in thread_lock().

bsg_kscrash_setSearchThreadNames

void bsg_kscrash_setSearchThreadNames(
    bool shouldSearchThreadNames);
Discussion

If true, search for thread names where appropriate. Thread name searching is not async-safe, and so comes with the risk of timing out and panicking in thread_lock().

bsg_kscrash_setSuspendThreadsForUserReported

void bsg_kscrash_setSuspendThreadsForUserReported( 
    bool suspendThreadsForUserReported);
Discussion

If YES, user reported exceptions will suspend all threads during report generation. All threads will be suspended while generating a crash report for a user reported exception.

Default: YES
