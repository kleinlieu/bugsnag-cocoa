redirectConsoleLogsToDefaultFile

- (BOOL)redirectConsoleLogsToDefaultFile;
Return Value

true if the operation was successful.

Discussion

Sets logFilePath to the default log file location (Library/Caches/KSCrashReports/<bundle name>-CrashLog.txt). If the file exists, it will be overwritten.

redirectConsoleLogsToFile:overwrite:

- (BOOL)redirectConsoleLogsToFile:(NSString *)fullPath 
        overwrite:(BOOL)overwrite;
Parameters
fullPath

The path to the logfile (nil = log to console instead).

overwrite

If true, overwrite the file (ignored if fullPath is nil).

Return Value

true if the operation was successful.

Discussion

Redirect the log of BSG_KSCrash's activities from the console to the specified log file.

reportCount

- (NSUInteger)reportCount;
Discussion

The total number of unsent reports. Note: This is an expensive operation.

onCompletion

logFilePath

@property(nonatomic,
    readonly,
    retain) NSString *logFilePath;
Discussion

Path where the log of BSG_KSCrash's activities will be written. If nil, log entries will be printed to the console.

This property cannot be set directly. Use one of the "redirectConsoleLogs" methods instead.

Default: nil

maxStoredReports

Discussion

Max number of reports to store on disk before throwing older reports out. (default 5)

onCrash

@property(nonatomic,
    readwrite,
    assign) BSGReportCallback onCrash;
Discussion

C Function to call during a crash report to give the callee an opportunity to add to the report. NULL = ignore.

WARNING: Only call async-safe functions from this function! DO NOT call Objective-C methods!!!

Note: If you use an installation, it will automatically set this property. Do not modify it in such a case.

@property(nonatomic,
    readwrite,
    retain) id<BSG_KSCrashReportFilter> sink;
Discussion

The report sink where reports get sent. This MUST be set or else the reporter will not send reports (although it will still record them).
